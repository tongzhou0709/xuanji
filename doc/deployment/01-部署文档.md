# 部署文档 - 前后端分离部署

## 1. 部署架构

### 1.1 架构图

```
                    ┌─────────────────┐
                    │   用户浏览器     │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │   前端服务器     │
                    │   (Nginx)       │
                    │   - Vue静态文件  │
                    │   - 反向代理     │
                    └────────┬────────┘
                             │ HTTP/HTTPS
                    ┌────────▼────────┐
                    │   后端服务器     │
                    │   (SpringBoot)  │
                    │   - REST API     │
                    │   - 业务逻辑     │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │   数据库服务器   │
                    │   (MySQL)       │
                    └─────────────────┘
```

### 1.2 服务器配置要求

#### 前端服务器
- **操作系统**：Linux (CentOS 7+/Ubuntu 18.04+) 或 Windows Server
- **Web服务器**：Nginx 1.18+
- **Node.js**：14.0+ (仅用于构建)
- **磁盘空间**：至少 5GB
- **内存**：至少 2GB
- **网络**：需要开放 80/443 端口

#### 后端服务器
- **操作系统**：Linux (CentOS 7+/Ubuntu 18.04+) 或 Windows Server
- **JDK**：JDK 8 或 JDK 11
- **磁盘空间**：至少 10GB
- **内存**：至少 4GB
- **网络**：需要开放应用端口（默认 8080）

#### 数据库服务器
- **操作系统**：Linux (CentOS 7+/Ubuntu 18.04+) 或 Windows Server
- **MySQL**：MySQL 8.0+
- **磁盘空间**：至少 50GB
- **内存**：至少 4GB
- **网络**：需要开放 3306 端口

## 2. 环境准备

### 2.1 前端服务器环境准备

#### 2.1.1 安装 Nginx

**Linux (CentOS):**
```bash
# 安装 EPEL 源
yum install epel-release -y

# 安装 Nginx
yum install nginx -y

# 启动 Nginx
systemctl start nginx

# 设置开机自启
systemctl enable nginx
```

**Linux (Ubuntu):**
```bash
# 更新软件包列表
apt update

# 安装 Nginx
apt install nginx -y

# 启动 Nginx
systemctl start nginx

# 设置开机自启
systemctl enable nginx
```

**Windows:**
1. 下载 Nginx for Windows: http://nginx.org/en/download.html
2. 解压到指定目录（如 C:\nginx）
3. 运行 nginx.exe

#### 2.1.2 安装 Node.js（仅用于构建）

**Linux:**
```bash
# 下载 Node.js
wget https://nodejs.org/dist/v14.17.0/node-v14.17.0-linux-x64.tar.xz

# 解压
tar -xvf node-v14.17.0-linux-x64.tar.xz

# 配置环境变量
echo 'export PATH=/path/to/node-v14.17.0-linux-x64/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

# 验证安装
node -v
npm -v
```

**Windows:**
1. 下载 Node.js 安装包: https://nodejs.org/
2. 运行安装程序
3. 验证安装：打开命令行输入 `node -v` 和 `npm -v`

### 2.2 后端服务器环境准备

#### 2.2.1 安装 JDK

**Linux (CentOS):**
```bash
# 下载 JDK
wget https://download.oracle.com/java/8/latest/jdk-8u301-linux-x64.tar.gz

# 解压
tar -xzf jdk-8u301-linux-x64.tar.gz -C /usr/local/

# 配置环境变量
echo 'export JAVA_HOME=/usr/local/jdk1.8.0_301' >> ~/.bashrc
echo 'export PATH=$JAVA_HOME/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

# 验证安装
java -version
```

**Linux (Ubuntu):**
```bash
# 安装 OpenJDK
apt install openjdk-8-jdk -y

# 验证安装
java -version
```

**Windows:**
1. 下载 JDK 安装包: https://www.oracle.com/java/technologies/downloads/
2. 运行安装程序
3. 配置环境变量 JAVA_HOME 和 PATH

#### 2.2.2 创建应用目录

```bash
# 创建应用目录
mkdir -p /opt/xuanji/backend
mkdir -p /opt/xuanji/logs

# 创建配置目录
mkdir -p /opt/xuanji/config
```

### 2.3 数据库服务器环境准备

#### 2.3.1 安装 MySQL

**Linux (CentOS):**
```bash
# 下载 MySQL Yum 源
wget https://dev.mysql.com/get/mysql80-community-release-el7-3.noarch.rpm

# 安装 Yum 源
rpm -ivh mysql80-community-release-el7-3.noarch.rpm

# 安装 MySQL
yum install mysql-server -y

# 启动 MySQL
systemctl start mysqld

# 设置开机自启
systemctl enable mysqld

# 获取临时密码
grep 'temporary password' /var/log/mysqld.log
```

**Linux (Ubuntu):**
```bash
# 更新软件包列表
apt update

# 安装 MySQL
apt install mysql-server -y

# 启动 MySQL
systemctl start mysql

# 设置开机自启
systemctl enable mysql
```

**Windows:**
1. 下载 MySQL 安装包: https://dev.mysql.com/downloads/mysql/
2. 运行安装程序
3. 配置 root 密码

#### 2.3.2 创建数据库

```sql
-- 登录 MySQL
mysql -u root -p

-- 创建数据库
CREATE DATABASE xuanji DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建用户
CREATE USER 'xuanji'@'%' IDENTIFIED BY 'Xuanji@123456';

-- 授权
GRANT ALL PRIVILEGES ON xuanji.* TO 'xuanji'@'%';

-- 刷新权限
FLUSH PRIVILEGES;

```

## 3. 应用部署

### 3.1 后端部署

#### 3.1.1 打包应用

在开发环境中执行：

```bash
# 进入后端项目目录
cd xuanji-backend

# 打包（跳过测试）
mvn clean package -DskipTests

# 生成的 JAR 包位置
# xuanji-backend/target/xuanji-backend-1.0.0.jar
```

#### 3.1.2 上传 JAR 包

```bash
# 使用 scp 上传
scp xuanji-backend/target/xuanji-backend-1.0.0.jar user@backend-server:/opt/xuanji/backend/
```

#### 3.1.3 配置文件

创建配置文件 `/opt/xuanji/config/application-prod.yml`:

```yaml
server:
  port: 8080
  servlet:
    context-path: /api

spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://数据库服务器IP:3306/xuanji?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false
    username: xuanji
    password: Xuanji@123456
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      connection-timeout: 30000

  jackson:
    date-format: yyyy-MM-dd HH:mm:ss
    time-zone: GMT+8

mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
  global-config:
    db-config:
      id-type: auto
      logic-delete-field: deleted
      logic-delete-value: 1
      logic-not-delete-value: 0

logging:
  level:
    com.xuanji: debug
  file:
    name: /opt/xuanji/logs/xuanji-backend.log
  pattern:
    file: '%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{50} - %msg%n'
```

#### 3.1.4 创建启动脚本

创建启动脚本 `/opt/xuanji/backend/start.sh`:

```bash
#!/bin/bash

APP_NAME="xuanji-backend-1.0.0.jar"
APP_DIR="/opt/xuanji/backend"
LOG_DIR="/opt/xuanji/logs"
CONFIG_DIR="/opt/xuanji/config"

# 停止旧进程
PID=$(ps -ef | grep $APP_NAME | grep -v grep | awk '{print $2}')
if [ -n "$PID" ]; then
    echo "Stopping $APP_NAME (PID: $PID)..."
    kill -15 $PID
    sleep 5
fi

# 启动新进程
echo "Starting $APP_NAME..."
nohup java -jar $APP_DIR/$APP_NAME \
    --spring.config.location=$CONFIG_DIR/application-prod.yml \
    > $LOG_DIR/xuanji-backend.out 2>&1 &

echo "Application started successfully!"
```

赋予执行权限：
```bash
chmod +x /opt/xuanji/backend/start.sh
```

#### 3.1.5 创建停止脚本

创建停止脚本 `/opt/xuanji/backend/stop.sh`:

```bash
#!/bin/bash

APP_NAME="xuanji-backend-1.0.0.jar"

PID=$(ps -ef | grep $APP_NAME | grep -v grep | awk '{print $2}')
if [ -n "$PID" ]; then
    echo "Stopping $APP_NAME (PID: $PID)..."
    kill -15 $PID
    sleep 5
    echo "Application stopped successfully!"
else
    echo "Application is not running!"
fi
```

赋予执行权限：
```bash
chmod +x /opt/xuanji/backend/stop.sh
```

#### 3.1.6 启动应用

```bash
# 启动应用
/opt/xuanji/backend/start.sh

# 查看日志
tail -f /opt/xuanji/logs/xuanji-backend.out

# 验证启动
curl http://localhost:8080/api/health
```

#### 3.1.7 配置防火墙

```bash
# 开放 8080 端口
firewall-cmd --zone=public --add-port=8080/tcp --permanent
firewall-cmd --reload
```

### 3.2 前端部署

#### 3.2.1 构建应用

在开发环境中执行：

```bash
# 进入前端项目目录
cd xuanji-frontend

# 安装依赖
npm install

# 修改生产环境配置
# 修改 .env.production 文件
# VUE_APP_BASE_API=http://后端服务器IP:8080/api

# 构建
npm run build

# 生成的静态文件在 dist 目录
```

#### 3.2.2 上传静态文件

```bash
# 压缩 dist 目录
cd dist
tar -czf dist.tar.gz *

# 上传到前端服务器
scp dist.tar.gz user@frontend-server:/opt/xuanji/frontend/

# 在前端服务器上解压
ssh user@frontend-server
cd /opt/xuanji/frontend
tar -xzf dist.tar.gz -C /opt/xuanji/frontend/
```

#### 3.2.3 配置 Nginx

创建 Nginx 配置文件 `/etc/nginx/conf.d/xuanji.conf`:

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /opt/xuanji/frontend;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 反向代理到后端
    location /api/ {
        proxy_pass http://后端服务器IP:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        root /opt/xuanji/frontend;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript application/x-javascript application/xml+rss application/json application/javascript;
}
```

#### 3.2.4 测试配置

```bash
# 测试 Nginx 配置
nginx -t

# 如果配置正确，重新加载 Nginx
nginx -s reload
```

#### 3.2.5 配置防火墙

```bash
# 开放 80 端口
firewall-cmd --zone=public --add-port=80/tcp --permanent
firewall-cmd --reload
```

## 4. HTTPS 配置（可选）

### 4.1 申请 SSL 证书

使用 Let's Encrypt 免费证书：

```bash
# 安装 Certbot
yum install certbot python2-certbot-nginx -y

# 申请证书
certbot --nginx -d your-domain.com

# 自动续期
certbot renew --dry-run
```

### 4.2 修改 Nginx 配置

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # 其他配置保持不变
    location / {
        root /opt/xuanji/frontend;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://后端服务器IP:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}
```

### 5.3 备份策略

#### 数据库备份
```bash
# 创建备份脚本
cat > /opt/backup/mysql_backup.sh << 'EOF'
#!/bin/bash
BACKUP_DIR="/opt/backup/mysql"
DATE=$(date +%Y%m%d_%H%M%S)
MYSQL_USER="xuanji"
MYSQL_PASSWORD="Xuanji@123456"
DATABASE="xuanji"

mkdir -p $BACKUP_DIR

mysqldump -u$MYSQL_USER -p$MYSQL_PASSWORD $DATABASE | gzip > $BACKUP_DIR/xuanji_$DATE.sql.gz

# 删除 7 天前的备份
find $BACKUP_DIR -name "xuanji_*.sql.gz" -mtime +7 -delete
EOF

chmod +x /opt/backup/mysql_backup.sh

# 添加到 crontab
crontab -e
# 每天凌晨 2 点执行备份
0 2 * * * /opt/backup/mysql_backup.sh
```

#### 应用备份
```bash
# 备份 JAR 包
cp /opt/xuanji/backend/xuanji-backend-1.0.0.jar /opt/backup/xuanji-backend-1.0.0.jar

# 备份前端文件
tar -czf /opt/backup/frontend_$(date +%Y%m%d).tar.gz /opt/xuanji/frontend/
```

## 7. 更新部署

### 7.1 后端更新

```bash
# 1. 停止应用
/opt/xuanji/backend/stop.sh

# 2. 备份旧版本
cp /opt/xuanji/backend/xuanji-backend-1.0.0.jar /opt/xuanji/backend/xuanji-backend-1.0.0.jar.bak

# 3. 上传新版本
scp xuanji-backend/target/xuanji-backend-1.0.1.jar user@backend-server:/opt/xuanji/backend/

# 4. 更新启动脚本中的 JAR 包名称
vi /opt/xuanji/backend/start.sh

# 5. 启动新版本
/opt/xuanji/backend/start.sh

# 6. 验证
curl http://localhost:8080/api/health
```

### 7.2 前端更新

```bash
# 1. 构建新版本
cd xuanji-frontend
npm run build

# 2. 备份旧版本
ssh user@frontend-server
mv /opt/xuanji/frontend /opt/xuanji/frontend.bak

# 3. 上传新版本
scp -r dist/* user@frontend-server:/opt/xuanji/frontend/

# 4. 清理缓存（可选）
ssh user@frontend-server
rm -rf /opt/xuanji/frontend/static/*
```
