# 用户故事管理说明

## 目录结构

```
用户故事/
├── 02-基础管理模块.md    # 基础管理模块用户故事
├── 03-任务管理模块.md    # 任务管理模块用户故事
└── README.md            # 本文件
```

## 模块划分

### 基础管理模块
包含系统的核心基础功能：
- 登录认证
- 用户管理
- 机构管理
- 菜单管理
- 角色管理
- 权限控制
- 系统风格与布局

### 任务管理模块
包含任务相关功能：
- 重点任务管理

## 用户故事编号规则

- **基础管理模块**：US-001 ~ US-099
- **任务管理模块**：US-100 ~ US-199
- 其他模块：US-200 ~ US-999

## 用户故事状态

- **待开发**：尚未开始开发
- **开发中**：正在开发中
- **已完成**：已完成开发
- **已废弃**：不再需要此功能

## 使用规范

### 新增功能
1. 在对应的模块文件中添加新的用户故事
2. 使用统一的用户故事格式
3. 分配正确的用户故事编号
4. 设置合理的优先级和状态

### 修改功能
1. 在已有的用户故事上修改，不要创建新的用户故事
2. 更新验收标准
3. 更新状态
4. 记录修改历史

### 用户故事格式

```markdown
### US-XXX 用户故事标题
- **作为**：角色
- **我想要**：需求
- **以便**：价值
- **验收标准**：
  - 验收条件1
  - 验收条件2
- **优先级**：高/中/低
- **状态**：待开发/开发中/已完成
```

## 版本管理

每个用户故事文档底部包含：
- 文档版本号
- 创建日期
- 最后更新日期

每次修改文档时，请更新版本号和最后更新日期。
