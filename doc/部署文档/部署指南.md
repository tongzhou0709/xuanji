# 部署文档 - 前后端分离部署

## 1. 部署架构

### 1.1 架构图

```
                    ┌─────────────────┐
                    │   用户浏览器     │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │   前端服务器     │
                    │   (Nginx)       │
                    │   - Vue静态文件  │
                    │   - 反向代理     │
                    └────────┬────────┘
                             │ HTTP/HTTPS
                    ┌────────▼────────┐
                    │   后端服务器     │
                    │   (Tomcat)      │
                    │   - WAR包部署   │
                    │   - REST API     │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │   数据库服务器   │
                    │   (MySQL 8.0+)  │
                    └─────────────────┘
```

### 1.2 服务器配置要求

#### 前端服务器
- **操作系统**：Linux (CentOS 7+/Ubuntu 18.04+) 或 Windows Server
- **Web服务器**：Nginx 1.18+
- **Node.js**：14.0+ (仅用于构建)
- **磁盘空间**：至少 5GB
- **内存**：至少 2GB
- **网络**：需要开放 80/443 端口

#### 后端服务器
- **操作系统**：Linux (CentOS 7+/Ubuntu 18.04+) 或 Windows Server
- **JDK**：JDK 8 (必须使用JDK 8)
- **Tomcat**：Tomcat 9.0.105+
- **磁盘空间**：至少 10GB
- **内存**：至少 4GB
- **网络**：需要开放 8080 端口

#### 数据库服务器
- **操作系统**：Linux (CentOS 7+/Ubuntu 18.04+) 或 Windows Server
- **MySQL**：MySQL 8.0+
- **磁盘空间**：至少 50GB
- **内存**：至少 4GB
- **网络**：需要开放 3306 端口

## 2. 环境准备

### 2.1 前端服务器环境准备

#### 2.1.1 安装 Nginx

**Linux (CentOS):**
```bash
# 安装 EPEL 源
yum install epel-release -y

# 安装 Nginx
yum install nginx -y

# 启动 Nginx
systemctl start nginx

# 设置开机自启
systemctl enable nginx
```

**Linux (Ubuntu):**
```bash
# 更新软件包列表
apt update

# 安装 Nginx
apt install nginx -y

# 启动 Nginx
systemctl start nginx

# 设置开机自启
systemctl enable nginx
```

**Windows:**
1. 下载 Nginx for Windows: http://nginx.org/en/download.html
2. 解压到指定目录（如 C:\nginx）
3. 运行 nginx.exe

#### 2.1.2 安装 Node.js（仅用于构建）

**Linux:**
```bash
# 下载 Node.js
wget https://nodejs.org/dist/v14.17.0/node-v14.17.0-linux-x64.tar.xz

# 解压
tar -xvf node-v14.17.0-linux-x64.tar.xz

# 配置环境变量
echo 'export PATH=/path/to/node-v14.17.0-linux-x64/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

# 验证安装
node -v
npm -v
```

**Windows:**
1. 下载 Node.js 安装包: https://nodejs.org/
2. 运行安装程序
3. 验证安装：打开命令行输入 `node -v` 和 `npm -v`

### 2.2 后端服务器环境准备

**Linux (CentOS):**

# 解压
tar -xzf jdk-8u301-linux-x64.tar.gz -C /usr/local/

**Linux (Ubuntu):**

**Windows:**
1. 下载 JDK 8 安装包: https://www.oracle.com/java/technologies/javase/javase8-archive-downloads.html
2. 运行安装程序
3. 配置环境变量 JAVA_HOME 和 PATH

#### 2.2.2 安装 Tomcat 9.0.105

wget https://archive.apache.org/dist/tomcat/tomcat-9/v9.0.105/bin/apache-tomcat-9.0.105.tar.gz

# 解压
tar -xzf apache-tomcat-9.0.105.tar.gz -C /opt/

mv /opt/apache-tomcat-9.0.105 /opt/tomcat

# 设置权限

# 启动 Tomcat

**Windows:**

## 3. 应用部署

### 3.1 后端部署

#### 3.1.1 打包应用

在开发环境中执行：

```bash
# 进入后端项目目录
cd xuanji-backend

# 设置 JAVA_HOME 为 JDK 8
set JAVA_HOME=C:\Users\周瞳0709\.jdks\corretto-1.8.0_482

# 打包（跳过测试）
mvn clean package -DskipTests

# 生成的 WAR 包位置
# xuanji-backend/target/xuanji-backend.war
```

#### 3.1.2 上传 WAR 包

```bash
# 使用 scp 上传
scp xuanji-backend/target/xuanji-backend.war user@backend-server:/opt/tomcat/webapps/
```

#### 3.1.3 配置数据库连接

编辑 Tomcat 的配置文件 `/opt/tomcat/conf/context.xml`，添加数据库连接池：

```xml
<Resource name="jdbc/xuanjiDB"
          auth="Container"
          type="javax.sql.DataSource"
          driverClassName="com.mysql.cj.jdbc.Driver"
          url="jdbc:mysql://数据库服务器IP:3306/xuanji?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false"
          username="xuanji"
          password="Xuanji@123456"
          maxTotal="20"
          maxIdle="5"
          maxWaitMillis="30000"
          validationQuery="SELECT 1"
          testOnBorrow="true"/>
```

或者直接修改 WAR 包内的配置文件：

1. 解压 WAR 包：
```bash
cd /opt/tomcat/webapps
unzip xuanji-backend.war -d xuanji-backend
```

2. 编辑配置文件 `/opt/tomcat/webapps/xuanji-backend/WEB-INF/classes/application-prod.yml`：

```yaml
server:
  port: 8080
  servlet:
    context-path: /xuanji-backend

spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://数据库服务器IP:3306/xuanji?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false
    username: xuanji
    password: Xuanji@123456
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      connection-timeout: 30000

  jackson:
    date-format: yyyy-MM-dd HH:mm:ss
    time-zone: GMT+8

mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
  global-config:
    db-config:
      id-type: auto
      logic-delete-field: deleted
      logic-delete-value: 1
      logic-not-delete-value: 0

logging:
  level:
    com.xuanji: debug
  file:
    name: /opt/xuanji/logs/xuanji-backend.log
  pattern:
    file: '%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{50} - %msg%n'
```

3. 重新打包：
```bash
cd /opt/tomcat/webapps
zip -r xuanji-backend.war xuanji-backend/*
rm -rf xuanji-backend
```

#### 3.1.4 配置 Tomcat

编辑 `/opt/tomcat/conf/server.xml`，确保端口配置正确：

```xml
<Server port="8005" shutdown="SHUTDOWN">
  <Service name="Catalina">
    <Connector port="8080" protocol="HTTP/1.1"
               connectionTimeout="20000"
               redirectPort="8443"
               maxThreads="200"
               minSpareThreads="10"/>
    <Engine name="Catalina" defaultHost="localhost">
      <Host name="localhost" appBase="webapps"
            unpackWARs="true" autoDeploy="true">
        <Context path="/xuanji-backend" docBase="xuanji-backend.war"
                 reloadable="false" crossContext="true"/>
      </Host>
    </Engine>
  </Service>
</Server>
```

#### 3.1.5 设置环境变量

编辑 `/opt/tomcat/bin/setenv.sh`：

```bash
#!/bin/bash
export JAVA_HOME=/usr/local/jdk1.8.0_301
export CATALINA_OPTS="-Xms512m -Xmx1024m -XX:PermSize=256m -XX:MaxPermSize=512m"
```

赋予执行权限：
```bash
chmod +x /opt/tomcat/bin/setenv.sh
```

#### 3.1.6 启动 Tomcat

```bash
# 启动 Tomcat
/opt/tomcat/bin/startup.sh

# 查看日志
tail -f /opt/tomcat/logs/catalina.out

# 验证启动
curl http://localhost:8080/xuanji-backend/health
```

#### 3.1.7 配置防火墙

```bash
# 开放 8080 端口
firewall-cmd --zone=public --add-port=8080/tcp --permanent
firewall-cmd --reload
```

### 3.2 前端部署

#### 3.2.1 构建应用

在开发环境中执行：

```bash
# 进入前端项目目录
cd xuanji-frontend

# 安装依赖
npm install

# 修改生产环境配置
# 编辑 .env.production 文件
# VUE_APP_BASE_API=http://后端服务器IP:8080/xuanji-backend

# 构建
npm run build

# 生成的静态文件在 dist 目录
```

#### 3.2.2 上传静态文件

```bash
# 压缩 dist 目录
cd dist
tar -czf dist.tar.gz *

# 上传到前端服务器
scp dist.tar.gz user@frontend-server:/opt/xuanji/frontend/

# 在前端服务器上解压
ssh user@frontend-server
cd /opt/xuanji/frontend
tar -xzf dist.tar.gz
```

#### 3.2.3 配置 Nginx

创建 Nginx 配置文件 `/etc/nginx/conf.d/xuanji.conf`:

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /opt/xuanji/frontend;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 反向代理到后端
    location /xuanji-backend/ {
        proxy_pass http://后端服务器IP:8080/xuanji-backend/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        root /opt/xuanji/frontend;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript application/x-javascript application/xml+rss application/json application/javascript;
}
```

#### 3.2.4 测试配置

```bash
# 测试 Nginx 配置
nginx -t

# 如果配置正确，重新加载 Nginx
nginx -s reload
```

#### 3.2.5 配置防火墙

```bash
# 开放 80 端口
firewall-cmd --zone=public --add-port=80/tcp --permanent
firewall-cmd --reload
```

## 4. HTTPS 配置（可选）

### 4.1 申请 SSL 证书

使用 Let's Encrypt 免费证书：

```bash
# 安装 Certbot
yum install certbot python2-certbot-nginx -y

# 申请证书
certbot --nginx -d your-domain.com

# 自动续期
certbot renew --dry-run
```

### 4.2 修改 Nginx 配置

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # 前端静态文件
    location / {
        root /opt/xuanji/frontend;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 反向代理到后端
    location /xuanji-backend/ {
        proxy_pass http://后端服务器IP:8080/xuanji-backend/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}
```

### 5.3 备份策略

#### 数据库备份
```bash
# 创建备份脚本
cat > /opt/backup/mysql_backup.sh << 'EOF'
#!/bin/bash
BACKUP_DIR="/opt/backup/mysql"
DATE=$(date +%Y%m%d_%H%M%S)
MYSQL_USER="xuanji"
MYSQL_PASSWORD="Xuanji@123456"
DATABASE="xuanji"

mkdir -p $BACKUP_DIR

mysqldump -u$MYSQL_USER -p$MYSQL_PASSWORD $DATABASE | gzip > $BACKUP_DIR/xuanji_$DATE.sql.gz

# 删除 7 天前的备份
find $BACKUP_DIR -name "xuanji_*.sql.gz" -mtime +7 -delete
EOF

chmod +x /opt/backup/mysql_backup.sh

# 添加到 crontab
crontab -e
# 每天凌晨 2 点执行备份
0 2 * * * /opt/backup/mysql_backup.sh
```

#### 应用备份
```bash
# 备份 WAR 包
cp /opt/tomcat/webapps/xuanji-backend.war /opt/backup/xuanji-backend_$(date +%Y%m%d).war

# 备份前端文件
tar -czf /opt/backup/frontend_$(date +%Y%m%d).tar.gz /opt/xuanji/frontend/
```
