# 事务管理系统 - 项目需求文档

## 1. 项目概述

### 1.1 项目背景
构建一个完整的事务管理系统，首先实现平台基础管理模块，包括用户管理、机构管理、菜单管理，并实现基于RBAC的权限控制系统。

### 1.2 项目目标
- 实现前后端分离架构
- 支持多机构、多用户管理
- 实现灵活的权限控制
- 支持跨服务器部署
- 提供良好的用户体验

### 1.3 技术选型
- 后端：SpringBoot 2.7+ + MyBatis-Plus + Spring Security + JWT
- 前端：Vue2 + Vue Router + Vuex + Element UI + Axios
- 数据库：MySQL 8.0+
- 缓存：Redis（可选）

## 2. 功能需求

### 2.1 用户管理模块

#### 2.1.1 功能描述
实现系统用户的增删改查功能，支持用户与机构的关联。

#### 2.1.2 功能列表
- 用户列表查询（支持分页、搜索）
- 用户新增（包含基本信息、所属机构、角色分配）
- 用户编辑（修改用户信息、调整角色）
- 用户删除（支持批量删除）
- 用户状态管理（启用/禁用）
- 密码重置
- 用户详情查看

#### 2.1.3 数据字段
- 用户ID
- 用户名（唯一）
- 密码（加密存储）
- 真实姓名
- 手机号
- 邮箱
- 所属机构ID
- 用户状态（正常/禁用）
- 创建时间
- 更新时间

### 2.2 机构管理模块

#### 2.2.1 功能描述
实现组织机构的树形结构管理，支持多级机构。

#### 2.2.2 功能列表
- 机构树形展示
- 机构新增（支持选择父机构）
- 机构编辑
- 机构删除（需检查是否有子机构和关联用户）
- 机构状态管理
- 机构排序

#### 2.2.3 数据字段
- 机构ID
- 机构名称
- 机构编码（唯一）
- 父机构ID
- 机构层级
- 排序号
- 机构状态（正常/禁用）
- 创建时间
- 更新时间

### 2.3 菜单管理模块

#### 2.3.1 功能描述
实现系统菜单的树形结构管理，支持动态路由生成。

#### 2.3.2 功能列表
- 菜单树形展示
- 菜单新增（支持目录、菜单、按钮三种类型）
- 菜单编辑
- 菜单删除
- 菜单排序
- 菜单权限标识配置

#### 2.3.3 数据字段
- 菜单ID
- 菜单名称
- 菜单类型（目录/菜单/按钮）
- 父菜单ID
- 路由路径
- 组件路径
- 菜单图标
- 权限标识
- 排序号
- 菜单状态（正常/禁用）
- 创建时间
- 更新时间

### 2.4 权限控制模块

#### 2.4.1 功能描述
基于RBAC模型实现权限控制，包括角色管理和权限分配。

#### 2.4.2 功能列表
- 角色列表查询
- 角色新增
- 角色编辑
- 角色删除
- 角色权限分配（菜单权限）
- 用户角色分配
- 权限验证（接口级、页面级、按钮级）

#### 2.4.3 数据字段
**角色表：**
- 角色ID
- 角色名称
- 角色编码（唯一）
- 角色描述
- 角色状态（正常/禁用）
- 创建时间
- 更新时间

**用户角色关联表：**
- 关联ID
- 用户ID
- 角色ID

**角色菜单关联表：**
- 关联ID
- 角色ID
- 菜单ID

### 2.5 登录认证模块

#### 2.5.1 功能描述
实现用户登录认证和Token管理。

#### 2.5.2 功能列表
- 用户登录（用户名/密码）
- Token生成和验证
- Token刷新
- 用户登出
- 登录状态保持
- 自动登录

## 3. 非功能需求

### 3.1 性能要求
- 页面响应时间 < 2秒
- 接口响应时间 < 500ms
- 支持100+并发用户

### 3.2 安全要求
- 密码加密存储（BCrypt）
- Token有效期管理
- 接口权限验证
- SQL注入防护
- XSS攻击防护

### 3.3 可用性要求
- 系统可用性 > 99%
- 支持主流浏览器（Chrome、Firefox、Edge）
- 响应式布局，支持不同分辨率

### 3.4 可维护性要求
- 代码规范统一
- 完善的注释
- 模块化设计
- 易于扩展

### 3.5 部署要求
- 支持前后端分离部署
- 支持跨服务器部署
- 配置外部化
- 支持Docker部署（可选）

## 4. 系统架构

### 4.1 整体架构
```
┌─────────────────┐         ┌─────────────────┐
│   前端服务器     │         │   后端服务器     │
│   (Vue2)        │         │   (SpringBoot)  │
│                 │         │                 │
│  - Vue Router   │◄────────┤  - REST API     │
│  - Vuex         │  HTTP   │  - Spring       │
│  - Element UI   │         │    Security     │
│  - Axios        │         │  - MyBatis-Plus │
└─────────────────┘         └────────┬────────┘
                                      │
                              ┌───────▼────────┐
                              │   数据库服务器   │
                              │   (MySQL)      │
                              └────────────────┘
```

### 4.2 后端架构
- Controller层：接口层，处理HTTP请求
- Service层：业务逻辑层
- Mapper层：数据访问层
- Security层：安全认证和授权
- Common层：公共组件（工具类、常量等）

### 4.3 前端架构
- Views层：页面视图
- Components层：公共组件
- Router层：路由配置
- Store层：状态管理
- API层：接口封装
- Utils层：工具函数

## 5. 接口规范

### 5.1 请求规范
- 使用RESTful风格
- Content-Type: application/json
- 认证方式：JWT Token（Header: Authorization: Bearer {token}）

### 5.2 响应规范
```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1234567890
}
```

### 5.3 状态码规范
- 200：成功
- 400：请求参数错误
- 401：未授权
- 403：无权限
- 404：资源不存在
- 500：服务器错误

## 6. 数据库设计

### 6.1 核心表
- sys_user：用户表
- sys_org：机构表
- sys_role：角色表
- sys_menu：菜单表
- sys_user_role：用户角色关联表
- sys_role_menu：角色菜单关联表

### 6.2 索引设计
- 用户名、机构编码、角色编码等唯一字段建立唯一索引
- 外键字段建立普通索引
- 查询频繁的字段建立复合索引

## 9. 部署要求

### 9.1 部署环境
- 前端服务器：Nginx + Vue静态文件
- 后端服务器：JDK 8+ + SpringBoot JAR包
- 数据库服务器：MySQL 8.0+

### 9.2 配置管理
- 使用配置文件外部化
- 敏感信息使用环境变量
- 支持多环境配置（dev、test、prod）

### 9.3 部署流程
1. 后端打包：mvn clean package
2. 前端打包：npm run build
3. 上传到对应服务器
4. 修改配置文件
5. 启动服务
6. 验证部署
