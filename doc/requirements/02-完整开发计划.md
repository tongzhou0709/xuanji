# 事务管理系统 - 完整开发计划

## 项目概述

构建一个完整的事务管理系统，实现平台基础管理模块，包括用户管理、机构管理、菜单管理，并实现基于RBAC的权限控制系统。系统采用前后端分离架构，支持跨服务器部署。

## 技术栈

### 后端技术栈
- **框架**：SpringBoot 2.7+
- **ORM**：MyBatis-Plus
- **安全**：Spring Security + JWT
- **数据库**：MySQL 8.0+
- **缓存**：Redis（可选）
- **构建工具**：Maven

### 前端技术栈
- **框架**：Vue2
- **路由**：Vue Router
- **状态管理**：Vuex
- **UI组件**：Element UI
- **HTTP客户端**：Axios
- **构建工具**：Vue CLI

### 部署技术栈
- **前端服务器**：Nginx
- **后端服务器**：JDK 8+
- **数据库服务器**：MySQL 8.0+
- **容器化**：Docker（可选）

## 开发规范

### 代码规范

#### 后端代码规范
- 遵循阿里巴巴Java开发手册
- 使用统一的命名规范
- 添加必要的注释
- 使用Lombok简化代码
- 使用MyBatis-Plus简化CRUD

#### 前端代码规范
- 遵循Vue官方风格指南
- 使用ESLint进行代码检查
- 使用Prettier进行代码格式化
- 组件命名使用PascalCase
- 文件命名使用kebab-case

### Git规范

#### 分支管理
- **master**：主分支，用于生产环境
- **develop**：开发分支，用于集成开发
- **feature/xxx**：功能分支，用于开发新功能
- **bugfix/xxx**：修复分支，用于修复Bug

#### 提交信息规范
- **feat**：新功能
- **fix**：修复Bug
- **docs**：文档更新
- **style**：代码格式调整
- **refactor**：重构
- **test**：测试
- **chore**：构建过程或辅助工具的变动

#### 提交信息格式
```
<type>(<scope>): <subject>

<body>

<footer>
```

### 文档规范

#### 开发日志
- 每次开发完成后更新开发日志
- 记录实现的功能

## 部署要求

### 可移植性要求

#### 配置外部化
- 所有配置项都支持外部化
- 敏感信息使用环境变量
- 支持多环境配置（dev、test、prod）

#### 路径配置
- 使用相对路径
- 支持自定义安装路径
- 日志文件路径可配置

#### 数据库配置
- 数据库连接信息可配置
- 支持不同数据库版本
- 数据库初始化脚本独立

### 部署架构

#### 前后端分离部署
- 前端和后端部署在不同服务器
- 通过Nginx反向代理
- 支持跨域访问

#### 容器化部署（可选）
- 提供Dockerfile
- 提供Docker Compose配置
- 支持一键部署

### 部署流程

#### 后端部署
1. 打包应用（mvn clean package）
2. 上传JAR包到服务器
3. 配置环境变量
4. 启动应用
5. 验证部署

#### 前端部署
1. 构建应用（npm run build）
2. 上传静态文件到服务器
3. 配置Nginx
4. 重启Nginx
5. 验证部署

#### 数据库部署
1. 创建数据库
2. 执行初始化脚本
3. 配置用户权限
4. 验证连接
