## 核心要求
1. 用户故事需含界面原型/效果描述、验收标准
2. 每次部署更新变更日志（v[X.X.X]/变更日志.md）
3. 数据库变更必须创建SQL脚本（db/update_*.sql）
4. Vue Router 锁定 2.8.0 版本
5. 后端代码必须 Java 8 兼容
6. 编写前端代码时需要保证风格一致
7. 每次开发完成后必须进行测试，针对所有边界条件进行测试，包括但不限于：
    - 正常输入
    - 异常输入（空值、无效格式、超出范围等）
    - 边界条件（最小值、最大值、临界值等）
    - 并发访问（多用户同时操作）
    - 性能测试（响应时间、吞吐量等）

## 目录结构
- doc/用户故事/ - 用户故事
- 部署/vX.X.X/ - 版本部署包
- xuanji-backend/src/main/resources/db/ - 数据库脚本
