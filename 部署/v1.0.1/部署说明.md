# 部署说明 - v1.0.1

## 版本信息
- **版本号**: v1.0.1
- **发布日期**: 2026-03-03
- **部署环境**: 生产环境 (prod)

## 部署文件清单

### 后端文件
- `xuanji-backend.war` - Spring Boot WAR 包
- **Context-path**: `/xuanji-backend`
- **端口**: 8080
- **JDK要求**: JDK 8

### 前端文件
- `dist/` - Vue 静态文件目录
- **API地址**: `/xuanji-backend` (相对路径，由Nginx代理)
- **路由模式**: hash

### 数据库文件
- `database_backup_*.sql` - 数据库备份文件
- **字符集**: utf8mb4

## 部署步骤

### 1. 后端部署

#### 1.1 上传WAR包
```bash
# 上传到Tomcat webapps目录
scp xuanji-backend.war user@backend-server:/opt/tomcat/webapps/
```

#### 1.2 配置数据库连接
编辑 `/opt/tomcat/webapps/xuanji-backend/WEB-INF/classes/application-prod.yml`:

```yaml
spring:
  datasource:
    url: jdbc:mysql://数据库服务器IP:6446/xj_dev_db?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false
    username: xj_admin
    password: 123456Qq!
```

#### 1.3 启动Tomcat
```bash
# 停止Tomcat
/opt/tomcat/bin/shutdown.sh

# 删除旧版本
rm -rf /opt/tomcat/webapps/xuanji-backend
rm -rf /opt/tomcat/work/Catalina/localhost/xuanji-backend

# 启动Tomcat
/opt/tomcat/bin/startup.sh

# 查看日志
tail -f /opt/tomcat/logs/catalina.out
```

#### 1.4 验证部署
```bash
# 检查应用是否启动
curl http://localhost:8080/xuanji-backend/health
```

### 2. 前端部署

#### 2.1 上传静态文件
```bash
# 压缩dist目录
cd dist
tar -czf dist.tar.gz *

# 上传到前端服务器
scp dist.tar.gz user@frontend-server:/opt/xuanji/frontend/

# 在前端服务器上解压
ssh user@frontend-server
cd /opt/xuanji/frontend
tar -xzf dist.tar.gz
```

#### 2.2 配置Nginx
创建或编辑 `/etc/nginx/conf.d/xuanji.conf`:

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /opt/xuanji/frontend;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 反向代理到后端
    location /xuanji-backend/ {
        proxy_pass http://后端服务器IP:8080/xuanji-backend/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        root /opt/xuanji/frontend;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript application/x-javascript application/xml+rss application/json application/javascript;
}
```

#### 2.3 测试配置
```bash
# 测试Nginx配置
nginx -t

# 重新加载Nginx
nginx -s reload
```

### 3. 数据库部署

#### 3.1 导入数据库
```bash
# 上传SQL文件
scp database_backup_*.sql user@database-server:/tmp/

# 登录数据库服务器
ssh user@database-server

# 导入数据库
mysql -h 127.0.0.1 -P 6446 -u xj_admin -p xj_dev_db < /tmp/database_backup_*.sql
```

#### 3.2 验证数据库
```sql
-- 检查表是否创建成功
SHOW TABLES;

-- 检查数据是否正确
SELECT COUNT(*) FROM sys_user;
```

## 重要配置说明

### Context-path

- 前端API: `/xuanji-backend` (相对路径)

### JDK版本

### 数据库配置

- 不需要服务器路由配置

## 故障排查

### 后端无法启动
1. 检查JDK版本: `java -version` (必须是1.8.x)
2. 检查端口占用: `netstat -tlnp | grep 8080`
3. 查看日志: `tail -f /opt/tomcat/logs/catalina.out`
4. 检查数据库连接

### 前端无法访问
1. 检查Nginx状态: `systemctl status nginx`
2. 检查Nginx配置: `nginx -t`
3. 检查防火墙: `firewall-cmd --list-all`

### 接口调用失败
1. 检查后端日志
2. 检查网络连通性
3. 检查Context-path配置

## 回滚方案

### 后端回滚
```bash
# 停止Tomcat
/opt/tomcat/bin/shutdown.sh

# 恢复旧版本WAR包
cp /opt/backup/xuanji-backend_v1.0.0.war /opt/tomcat/webapps/xuanji-backend.war

# 删除当前版本
rm -rf /opt/tomcat/webapps/xuanji-backend
rm -rf /opt/tomcat/work/Catalina/localhost/xuanji-backend

# 启动Tomcat
/opt/tomcat/bin/startup.sh
```

### 前端回滚
```bash
# 备份当前版本
mv /opt/xuanji/frontend /opt/xuanji/frontend.v1.0.1

# 恢复旧版本
mv /opt/xuanji/frontend.v1.0.0 /opt/xuanji/frontend
```
