## 部署流程

### 开发环境打包步骤

#### 1. 打包后端
```bash
# 进入后端项目目录
cd xuanji-backend

# 使用Maven打包WAR包
mvn clean package -DskipTests

# 复制WAR包到部署目录
cp target/xuanji-backend.war ../部署/v1.0.0/
```

#### 2. 打包前端
```bash
# 进入前端项目目录
cd xuanji-frontend

# 安装依赖
npm install

# 构建生产版本
npm run build

# 复制dist目录到部署目录
cp -r dist ../部署/v1.0.0/
```

#### 3. 生成数据库备份
```bash
# 导出数据库
mysqldump -h localhost -P 3306 -u root -pzt20020709 xj_dev > ../部署/v1.0.0/xj_dev_backup.sql

# 验证中文字符
file ../部署/v1.0.0/xj_dev_backup.sql
```

#### 4. 部署到Docker测试
```bash
# 停止旧容器
docker stop xuanji-backend-jdk11
docker rm -f xuanji-backend-jdk11

# 启动新容器
docker run -d \
  --name xuanji-backend-jdk11 \
  -p 8080:8080 \
  -v $(pwd)/部署/v1.0.0/xuanji-backend.war:/usr/local/tomcat/webapps/xuanji-backend.war \
  tomcat:9.0.105-jdk11

# 测试登录接口
curl -X POST http://localhost:8080/xuanji-backend/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
```

## 后端部署步骤

### 1. 准备工作
```bash
# 上传WAR包到服务器
scp xuanji-backend.war user@backend-server:/opt/deploy/
```

### 2. 使用Docker部署后端

#### 方法一：使用Docker命令直接部署
```bash
# 停止旧容器（如果存在）
docker stop xuanji-backend-jdk11
docker rm -f xuanji-backend-jdk11

# 启动新容器
docker run -d \
  --name xuanji-backend-jdk11 \
  -p 8080:8080 \
  -v /opt/deploy/xuanji-backend.war:/usr/local/tomcat/webapps/xuanji-backend.war \
  tomcat:9.0.105-jdk11
```

#### 方法二：使用Docker Compose部署
创建 `docker-compose.yml` 文件：
```yaml
version: '3'
services:
  xuanji-backend:
    image: tomcat:9.0.105-jdk11
    container_name: xuanji-backend-jdk11
    ports:
      - "8080:8080"
    volumes:
      - ./xuanji-backend.war:/usr/local/tomcat/webapps/xuanji-backend.war
    restart: unless-stopped
```

启动服务：
```bash
docker-compose up -d
```

### 3. 验证后端部署

#### 检查容器状态
```bash
# 查看容器运行状态
docker ps | grep xuanji-backend-jdk11

# 查看容器日志
```

#### 测试API接口

## 前端部署步骤

### 1. 上传前端文件
```bash
# 上传dist目录到前端服务器
scp -r dist/* user@frontend-server:/var/www/html/xuanji/
```

### 2. 配置Nginx反向代理

#### 创建Nginx配置文件
创建 `/etc/nginx/conf.d/xuanji.conf`：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /var/www/html/xuanji;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端API代理
    location /xuanji-backend/ {
        proxy_pass http://backend-server:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket支持（如果需要）
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        root /var/www/html/xuanji;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

### 3. 重启Nginx
```bash
# 测试配置文件
nginx -t

# 重启Nginx
systemctl restart nginx

# 或重新加载配置
nginx -s reload
```

### 4. 验证前端部署

#### 检查静态文件
```bash

curl http://your-domain.com/

# 检查index.html
curl -I http://your-domain.com/index.html
```

#### 测试API代理

```bash
  -H "Content-Type: application/json" \
```

## 常见问题排查

### 后端问题

#### 1. 容器无法启动
```bash
# 查看容器日志
docker logs xuanji-backend-jdk11

# 常见原因：
# - 端口被占用：修改映射端口
# - WAR包路径错误：检查volume挂载路径
# - 内存不足：增加Docker内存限制
```

#### 2. API返回404
```bash
# 检查WAR包是否正确部署
docker exec xuanji-backend-jdk11 ls -la /usr/local/tomcat/webapps/

# 检查context-path配置
# 默认context-path为/xuanji-backend
```

#### 3. 数据库连接失败
```bash
# 检查数据库连接
docker exec xuanji-backend-jdk11 ping -c 3 10.2.14.17

# 检查MySQL端口
telnet 10.2.14.17 6446

# 检查防火墙规则
firewall-cmd --list-ports
```

### 数据库问题

#### 1. 中文字符乱码
```bash
# 确保数据库字符集为utf8mb4
SHOW VARIABLES LIKE 'character_set%';

# 确保表字符集为utf8mb4
SHOW CREATE TABLE key_task;
```

#### 2. 导入失败
```bash
# 检查SQL文件编码
file xj_dev_backup.sql

# 使用指定编码导入
mysql --default-character-set=utf8mb4 < xj_dev_backup.sql
```

### 前端问题

#### 1. 静态资源404
```bash
# 检查Nginx root路径
nginx -T | grep root

# 检查文件权限
ls -la /var/www/html/xuanji
```

#### 2. API代理失败
```bash
# 检查后端服务器可访问性
curl http://backend-server:8080/xuanji-backend/auth/login

# 检查Nginx错误日志
tail -f /var/log/nginx/error.log
```
