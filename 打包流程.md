## 打包前检查

### 1. 检查后端配置

#### 1.1 检查application.yml的active profile
```bash
# 查看当前激活的profile
cat xuanji-backend/src/main/resources/application.yml | grep "active"
```

**预期结果**：
```yaml
spring:
  profiles:
    active: prod  # 必须是prod
```

**如果不是prod，需要修改**：
```yaml
spring:
  profiles:
    active: prod
```

#### 1.2 检查application-prod.yml配置
```bash
# 查看生产环境配置
cat xuanji-backend/src/main/resources/application-prod.yml
```

**关键配置检查**：
- Context-path: `/xuanji-backend`
- 数据库连接: 正确的生产数据库
- Redis配置: 正确的生产Redis（如果使用）

### 2. 检查WAR包是否存在

#### 2.1 检查WAR包
```bash
# 检查WAR包是否存在
Test-Path xuanji-backend/target/xuanji-backend.war
```

#### 2.2 检查WAR包时间戳
```bash
# 查看WAR包的最后修改时间
Get-Item xuanji-backend/target/xuanji-backend.war | Select-Object Name, Length, LastWriteTime
```

**如果WAR包不存在或时间戳不是最新的**：
⚠️ **提醒用户：后端WAR包不存在或不是最新的，需要先打包后端！**

**打包命令**：
```bash
cd xuanji-backend
./mvnw.cmd clean package -DskipTests
```

**注意**：
- 如果Maven wrapper的jar文件不存在，需要先下载或使用系统Maven
- 打包过程可能需要几分钟
- 打包成功后会在target目录生成xuanji-backend.war

### 3. 检查前端配置

#### 3.1 检查vue.config.js配置
```bash
# 查看vue.config.js配置
cat xuanji-frontend/vue.config.js
```

**关键配置检查**：
- publicPath: `process.env.NODE_ENV === 'production' ? './' : '/'`
- CSS loader配置: 包含全局SCSS变量配置

#### 3.2 检查.env.production配置
```bash
# 查看生产环境配置
cat xuanji-frontend/.env.production
```

**关键配置检查**：
- VUE_APP_BASE_API: `/xuanji-backend`

## 打包步骤

### 步骤1: 检查后端配置

```bash
# 检查application.yml的active profile
cat xuanji-backend/src/main/resources/application.yml | grep "active"

# 如果不是prod，修改为prod
# spring:
#   profiles:
#     active: prod
```

**确认**：
- [ ] active profile是prod
- [ ] application-prod.yml配置正确

### 步骤2: 检查后端WAR包

```bash
# 检查WAR包是否存在
Test-Path xuanji-backend/target/xuanji-backend.war

# 查看WAR包的最后修改时间
Get-Item xuanji-backend/target/xuanji-backend.war | Select-Object Name, Length, LastWriteTime
```

**如果WAR包不存在或时间戳不是最新的**：
⚠️ **提醒用户：后端WAR包不存在或不是最新的，需要先打包后端！**

**打包命令**：
```bash
cd xuanji-backend
./mvnw.cmd clean package -DskipTests
```

**确认**：
- [ ] WAR包存在
- [ ] WAR包时间戳是今天的
- [ ] WAR包大小正常（约68MB）

### 步骤3: 打包前端

```bash
# 进入前端目录
cd xuanji-frontend

# 打包前端（生产环境）
npm run build
```

**预期结果**：
- 打包时间：约20秒
- 生成目录：dist/
- 包含文件：index.html, static/css, static/js, static/fonts

**确认**：
- [ ] 打包成功（无致命错误）
- [ ] dist目录生成
- [ ] 包含所有必要文件

### 步骤4: 复制前端文件到部署目录

```bash
# 删除旧的dist目录
Remove-Item -Path "部署/v1.0.1/dist" -Recurse -Force

# 创建新的dist目录
New-Item -ItemType Directory -Path "部署/v1.0.1/dist" -Force

# 复制新的dist文件
Copy-Item -Path "xuanji-frontend/dist/*" -Destination "部署/v1.0.1/dist\" -Force
```

**确认**：
- [ ] 旧的dist目录已删除
- [ ] 新的dist文件已复制
- [ ] 部署目录中的dist是最新的

### 步骤5: 复制后端WAR包到部署目录

```bash
# 复制WAR包
Copy-Item -Path "xuanji-backend/target/xuanji-backend.war" -Destination "部署/v1.0.1\" -Force
```

**确认**：
- [ ] WAR包已复制到部署目录
- [ ] WAR包大小正常（约68MB）

### 步骤6: 验证部署文件完整性

```bash
# 查看部署目录内容
Get-ChildItem "部署/v1.0.1" | Select-Object Name, Mode, LastWriteTime

# 检查前端文件
Test-Path "部署/v1.0.1/dist/index.html"

# 检查后端文件
Test-Path "部署/v1.0.1/xuanji-backend.war"

# 检查部署文档
Test-Path "部署/v1.0.1/部署说明.md"
```

**预期结果**：
```
Name               Mode   LastWriteTime
----               ----   -------------
dist               d----- 2026-03-03 HH:MM:SS
xuanji-backend.war -a---- 2026-03-03 HH:MM:SS
部署说明.md        -a---- 2026-03-03 HH:MM:SS
```

**确认**：
- [ ] 部署目录包含dist、xuanji-backend.war、部署说明.md
- [ ] 所有文件的时间戳都是今天的
- [ ] 所有文件都可以访问

## 常见问题

### 1. 后端WAR包不存在

**症状**：检查WAR包时提示文件不存在

**原因**：后端没有打包或打包失败

**解决方案**：
```bash
cd xuanji-backend
./mvnw.cmd clean package -DskipTests
```

**注意**：
- 如果Maven wrapper的jar文件不存在，需要先下载或使用系统Maven
- 检查pom.xml中的Java版本是否为8
- 检查是否有编译错误

### 2. 后端配置不是prod

**症状**：application.yml中的active profile是dev

**原因**：忘记切换到生产环境配置

**解决方案**：
```yaml
# 修改application.yml
spring:
  profiles:
    active: prod
```

### 3. 前端打包失败

**症状**：npm run build报错

**原因**：代码错误、依赖问题、配置问题

**解决方案**：
```bash
# 清除缓存和依赖
cd xuanji-frontend
rm -rf node_modules
rm -rf package-lock.json
npm install

# 重新打包
npm run build
```

### 4. 前端样式问题

**症状**：打包后样式不正确

**原因**：SCSS变量配置问题

**解决方案**：
- 检查vue.config.js中的CSS loader配置
- 检查variables.scss中的变量定义
- 检查Sidebar组件中的颜色配置

### 5. 部署文件不完整

**症状**：部署目录缺少文件

**原因**：复制步骤出错

**解决方案**：
- 重新执行复制步骤
- 检查源文件是否存在
- 检查目标目录权限
